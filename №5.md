# Глава 5. ЛОГИ ПРЯМОГО ЭФИРА (PRESENT CONTINUOUS). ДИНАМИЧЕСКИЙ ПРОЦЕСС В ТОЧКЕ СИНХРОНИЗАЦИИ

## 1. Введение в архитектуру: Фиксация состояния в сис-минуту
В Главе 2 мы настроили регулярный конвейер процессов (`PRESENT SIMPLE`), который описывает задачи вообще, по регламенту (Cron). С этого рубежа мы разворачиваем инструмент **PRESENT CONTINUOUS** (Настоящее длительное). 

Этот инструмент задействуется, когда нам нужно зафиксировать **динамический процесс, который происходит прямо сейчас, в текущую миллисекунду времени**, непосредственно в момент генерации лога (в точке синхронизации).
* **Present Simple:** *«Я пишу скрипты на Python (вообще, это мой hard-скилл)»*.
* **Present Continuous:** *«Я пишу скрипт прямо сейчас (мои пальцы бьют по клавишам верстака в эту сис-минуту)»*.

В реальном разговоре на проекте этот инструмент незаменим для оперативных отчетов: *«Мы сейчас заливаем бетоном сектор Б»*, *«Сервер прямо сейчас перезагружается»*, *«Я сейчас проверяю твои правки в репозитории»*.

## 2. Инструмент №1: Движок живого процесса (Утверждение)

### Логика инструмента
Построение живой фразы в RAM. Конструкция CONTINUOUS — это гибрид, который собирается из двух элементов, уже знакомых твоей операционной системе:
1. **Системный оператор существования `TO BE`** (`am / is / are`) — выполняет роль несущей частоты, показывает *кто* находится в процессе прямо сейчас.
2. **Глагол-действие с принудительным суффиксом `-ING`** — переплавляет глагол в форму «причастия» (делающий / совершающий).

### Синтаксис
\[\text{[ КТО ]} \longrightarrow \mathbf{[ am / is / are ]} \longrightarrow \text{[ ГЛАГОЛ + ING ]} \longrightarrow \text{[ НАД КЕМ/ЧЕМ ]}\]

```text
I AM runnING a python script right now. (Я есмь запускающий питон-скрипт прямо сейчас)
The server IS uploadING the database. (Сервер есть загружающий базу данных)
We ARE checkING the welding logs. (Мы есть проверяющие сварочные логи)
```

### Физика работы
Система связывает исполнителя с его текущим состоянием через `TO BE`, а суффикс `-ING` вешает на глагол метку непрерывного движения. Это точный аналог лога в реальном времени, который пишет в консоль: `[PROCESSING... 45%]`.

## 3. Инструмент №2: Модификаторы отрицания и вопроса в логах прямого эфира

### Логика инструмента
Изменение направления потока живого процесса: фиксация того, что процесс прямо сейчас НЕ происходит (отрицание), или входящий запрос о текущем статусе задачи (вопрос).

### Синтаксис Отрицания (Щит `NOT` на несущей частоте)
Поскольку в этой архитектуре уже присутствует сильный системный оператор `TO BE`, нам **не нужен** дополнительный процессор `DO/DOES`. Мы монтируем логический щит `NOT` строго после `am / is / are`.
```text
I am NOT reviewING the code now, I am restING. (Я НЕ проверяю код сейчас, я отдыхаю)
The system is NOT processING files at the moment. (Система НЕ обрабатывает файлы в данный момент)
```

### Синтаксис Вопроса (Инверсия оператора существования)
Чтобы запустить входящий запрос о том, что происходит в текущую секунду, оператор `TO BE` (`am / is / are`) совершает стандартную рокировку и выносится на **самое первое место перед Исполнителем**.
\[\mathbf{[ Am / Is / Are ]} \longrightarrow \text{[ ИСПОЛНИТЕЛЬ ]} \longrightarrow \text{[ ГЛАГОЛ + ING ]} ?\]
```text
IS the contractor loadING the cement now? (Подрядчик сейчас загружает цемент?)
ARE you watchING the console logs? (Ты сейчас смотришь логи консоли?)
```

## 4. Инструмент №3: Статические ограничения — Капкан Глаголов Состояния (`STATIVE VERBS`)

### Логика инструмента
Это жесткий внутренний баг-фильтр архитектуры CONTINUOUS. Существует группа английских глаголов, которые обозначают не физическое действие, а внутренние процессы системы: умственную деятельность, владение, чувства или статические параметры. Они называются **Stative Verbs** (Глаголы состояния). 
* `know` (знать), `understand` (понимать), `want` (хотеть), `remember` (помнить), `have` (иметь/владеть).

Эти глаголы **физически запрещено использовать с суффиксом `-ING`** и загонять в структуру CONTINUOUS, даже если действие происходит прямо сейчас.

### Синтаксис (Защита Мастера)
Если тебе нужно сказать «Я понимаю тебя прямо сейчас», твой русский мозг требует применить Continuous. Но фильтр блокирует эту попытку и принудительно перенаправляет поток данных в чистый `PRESENT SIMPLE` из Главы 2.

```text
# КРИТИЧЕСКИЙ БАГ: I am understanding the SQL code now. (ОБРУШЕНИЕ СИСТЕМЫ)
# ИСПРАВЛЕНИЕ МАСТЕРА: I UNDERSTAND the SQL code now. (Я понимаю SQL-код сейчас)

# КРИТИЧЕСКИЙ БАГ: The system is having an error right now. (ОБРУШЕНИЕ СИСТЕМЫ)
# ИСПРАВЛЕНИЕ МАСТЕРА: The system HAS an error right now. (У системы сейчас ошибка)
```

### Защита Мастера
Загрузи в память базовый список запрещенных глаголов: `know, understand, want, love, like, remember, believe, have (в значении обладать)`. Как только на конвейер твоей мысли попадает один из этих глаголов — мгновенно блокируй суффикс `-ING`. Заставляй фразу звучать в простом настоящем времени, система собеседника сама достроит контекст «прямо сейчас».

## 5. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (ТРЕНАЖЕР ДЛЯ ТВОЕГО РАЗУМА)

Пропусти эти оперативные логи прямого эфира со строительной площадки и ИТ-верстака через свое ядро. Почувствуй разницу между фоновым регламентом (`Simple`) и живым процессом (`Continuous`).

### Кейс №1: Ситуационный отчет на ИТ-проекте (Исполнитель: Я / Мы)
*   **Утверждение:** `We ARE configurING the SQLAlchemy engine right now.` (Мы прямо сейчас настраиваем движок SQLAlchemy. *Режим реального времени*)
*   **Отрицание:** `I am NOT playING games, I am debuggING the Python code.` (Я НЕ играю в игры, я отлаживаю код на Python. *Щит NOT на месте*)
*   **Вопрос:** `ARE you exportING the data to the production server at this moment?` (Ты экспортируешь данные на рабочий сервер в этот момент? *ARE на передовой*)

### Кейс №2: Мониторинг автоматизации в ПТО (Исполнитель: Скрипт / Система)
*   **Утверждение:** `The system IS generatING the daily timesheet at this сис-минута.` (Система генерирует ежедневный табель в эту сис-минуту. *The system -> is -> generating*)
*   **Отрицание:** `The backup script is NOT runnING at the moment.` (Скрипт резервного копирования НЕ запущен в данный момент.)
*   **Вопрос:** `IS the database calculatING the total weight of cement?` (База данных сейчас рассчитывает общий вес цемента?)

### Кейс №3: Отработка капкана статических глаголов (Высший пилотаж защиты)
*   **Сценарий 1 (Понимание проблемы):** `I WANT to change the layout now.` (Я хочу изменить макет сейчас. *Глагол want заблокирован для -ING фильтром Stative Verbs, применен Simple*)
*   **Сценарий 2 (Фиксация памяти):** `The supervisor REMEMBER__S__ the limit rules.` (Начальник участка помнит правила лимитов. *Глагол remember в Simple, сработал капкан -S для третьего лица!*)
