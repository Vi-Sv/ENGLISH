# Глава 4. КОНВЕЙЕР РЕГУЛЯРНЫХ ДЕЙСТВИЙ (PRESENT SIMPLE)

## 1. Введение: Динамика и регулярность
Переход от статического `TO BE` к действиям. Present Simple описывает регулярные процессы, привычки и системные процедуры.
*   **Пример:** *We check logs daily.* (Мы проверяем логи ежедневно.)

---

## 2. Утверждение: Базовый глагол
Используется базовая форма глагола (без `to`).
*   **Схема:** `[Блок 1 (Кто)] + [Действие (Блок 2)]`
*   **Пример:** *I run updates.* (Я запускаю обновления.)

> **Важно:** Нельзя смешивать `TO BE` и глаголы-действия (*Incorrect: I am run*).

---

## 3. Закон третьего лица (-S / -ES)
Если в Блоке 1 — ОН, ОНА, ОНО (сервер, система), глагол получает окончание `-s`.
*   **Пример:** *The script start**s**.* / *She work**s**.*

---

## 4. Отрицание и Вопрос: Оператор «DO / DOES»
Для отрицания и вопросов нужны помощники: `DO` (для I, you, we, they) или `DOES` (для he, she, it).

### Отрицание (DO NOT / DOES NOT)
*   **Схема:** `[Субъект] + do/does + not + [Глагол]`
*   **Пример:** *I don't check it.* / *The system **doesn't** run [~~s~~].* (Важно: `DOES` забирает `-s`, у глагола его нет).

### Вопрос (DO / DOES)
*   **Схема:** `Do/Does + [Субъект] + [Глагол]?`
*   **Пример:** *Do you check logs?* / *Does the server fail?*

---

## 5. Практические примеры
*   **(+)** *We update the data.* (Мы обновляем данные.)
*   **(-)** *He **does not** update the data.* (Он не обновляет данные.)
*   **(?)** *Do they update the data?* (Они обновляют данные?)
