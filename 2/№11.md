# Глава 11. СИСТЕМНЫЕ ЗАПРОСЫ (СПЕЦИАЛЬНЫЕ ВОПРОСЫ): ИНСТРУМЕНТАРИЙ ТАРГЕТИРОВАНИЯ И ВЫБОРА

## 1. Введение: Адресация входящих запросов
В Главе 3 и последующих главах мы научились строить общие вопросы, которые требуют лишь бинарного ответа: `True` (Да) или `False` (Нет). 

Специальные вопросы (Wh-questions) используются для извлечения конкретных массивов данных из памяти собеседника. Для этого перед базовым вопросом разворачивается **Блок 0** — позиция для поисковых операторов (ключевых слов).

---

## 2. Инструментарий: Ключевые слова-таргеты (Блок 0)
Каждое вопросительное слово — это точечный селектор типа извлекаемой информации:

*   `WHAT` [wɒt] — селектор объектов или действий (Что? Какой?).
*   `WHERE` [weə] — селектор локации, адреса или дислокации (Где? Куда?).
*   `WHEN` [wen] — селектор временной метки или периода (Когда?).
*   `WHY` [waɪ] — селектор причины, логики или триггера (Зачем? Почему?).
*   `HOW` [haʊ] — селектор механизма, алгоритма или метода исполнения (Как?).
*   `WHO` [huː] — селектор исполнителя / субъекта (Кто?).

---

## 3. Архитектурный чертеж: Диктатура жесткого порядка выполнения (Query Order)
Попытка поставить русское вопросительное слово в хаотичную строку обрушит компиляцию. Английский специальный вопрос — это надстройка над уже готовым общим вопросом. Поисковый оператор ставится на нулевое место, не ломая структуру инверсии за ним.

### Синтаксическая сетка для динамических действий (Present / Past)
[ БЛОК 0: СЕЛЕКТОР ] ──> [ do / does / did ] ──> [ СУБЪЕКТ ] ──> [ ГЛАГОЛ В БАЗЕ ] ?
*   **Where do you store logs?** (Где ты хранишь логи? *Селектор WHERE + общий вопрос Do you store logs*)
*   **Why did the server crash?** (Почему сервер упал? *Селектор WHY + общий вопрос Did the server crash*)

### Синтаксическая сетка для статики и процессов (TO BE / Continuous)
[ БЛОК 0: СЕЛЕКТОР ] ──> [ am / is / are ] ──> [ СУБЪЕКТ ] ──> [ ХАРАКТЕРИСТИКА / ING-ГЛАГОЛ ] ?
*   **What is the status?** (Какой есть статус?)
*   **What are you debugging now?** (Что ты отлаживаешь сейчас?)

---

## 4. Исключение Мастера: Прямой запрос к исполнителю (Вопрос к подлежащему)
Если вы запрашиваете имя исполнителя через оператор `WHO` (Кто?), синтаксический конвейер резко меняется. В этом сценарии **запрещено** вызывать процессоры `DO / DOES / DID`. Оператор `WHO` сам встает на место Блока 1 (Субъекта), и предложение сохраняет структуру обычного плоского утверждения.

*   **Present Simple:** *Who **knows** the password?* (Кто знает пароль? *Глагол получает окончание -S, так как WHO по умолчанию считается единственным числом (It)*).
*   **Past Simple:** *Who **wrote** this code?* (Кто написал этот код? *Используется чистая форма V2*).

---

## 5. Практические примеры
*   **(?)** *When will you deploy the hotfix?* (Когда ты развернешь исправление?)
*   **(?)** *How has the team achieved this result?* (Как команда достигла этого результата? *Спец-вопрос в Present Perfect*)
*   **(?)** *Why is the pipeline blocked?* (Почему конвейер заблокирован? *Спец-вопрос в Пассивном залоге*)
*   **(?)** *Who approved the documentation yesterday?* (Кто утвердил документацию вчера? *Запрос к исполнителю, без DID*)
