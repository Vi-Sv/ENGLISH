# Глава 6. ВЕКТОР БУДУЩЕГО (FUTURE SIMPLE) И ПРЕДОХРАНИТЕЛЬ УСЛОВИЙ

## 1. Введение: Проектирование будущих тасков
Переход к планированию задач, прогнозам и спонтанным решениям в будущем времени. FUTURE SIMPLE — самый простой и стабильный инструмент, не требующий изменения окончаний.
*   **Пример:** *The system will send the report tomorrow.* (Система отправит отчет завтра.)

---

## 2. Утверждение: Скоростной оператор WILL
Для переброса моста в будущее перед базовым глаголом ставится неизменяемый оператор `WILL`.
*   **Схема:** `[Субъект] + will + [Глагол в базовой форме]`
*   **Пример:** *I will automate this workflow.* (Я автоматизирую этот рабочий процесс.)

> **Важно:** Глагол после `WILL` всегда остается чистым. Никаких окончаний `-s` или `-ed` внедрять нельзя.

---

## 3. Отрицание и Вопрос в архитектуре WILL
Оператор `WILL` обладает достаточной системной мощностью, чтобы самостоятельно держать щит отрицания и осуществлять рокировку.

### Отрицание (WILL NOT / WON'T)
В технической речи связка `will not` почти всегда компилируется в сжатую форму `WON'T` [wəʊnt].
*   **Пример:** *The server **won't** crash under heavy load.* (Сервер не упадет под высокой нагрузкой.)

### Вопрос (Инверсия оператора)
Для инициализации входящего запроса оператор `WILL` выносится на нулевую позицию перед субъектом.
*   **Пример:** ***Will** you deploy the code tonight?* (Ты развернешь код сегодня вечером?)

---

## 4. Предохранитель условий: Капкан IF / WHEN
Это главный логический барьер для русскоязычного мышления. В русском синтаксисе будущее время используется в обеих частях сложного предложения (*«Если скрипт упадет [буд.], я запущу [буд.] его вручную»*). 

В английской архитектуре ставить `WILL` внутри условных придаточных после операторов **IF (Если)** и **WHEN (Когда)** запрещено на уровне компилятора. Вместо будущего времени там принудительно активируется настоящее (`PRESENT SIMPLE` из Главы 4).

*   **Схема:** `IF / WHEN + [Present Simple (настоящее)] , [Субъект + WILL + Глагол (будущее)]`
*   **Пример:** *If the database **fails** [~~will fail~~], the script **will trigger** an alert.* (Если база данных упадет, скрипт запустит оповещение.)

> **Закон Мастера:** Помни про капкан окончания `-S` для третьего лица единственного числа в блоке настоящего времени после `IF/WHEN` (*The database fails / The supervisor signs*).

---

## 5. Практические примеры
*   **(+)** *We will write the pipeline next week.* (Мы напишем конвейер данных на следующей неделе.)
*   **(-)** *We **won't** waste time on manual sorting.* (Мы не будем тратить время на ручную сортировку.)
*   **(?)** ***Will** the contractor sign the document tomorrow?* (Подрядчик подпишет документ завтра?)
*   **(!)** *When I **finish** the dashboard, I **will upload** it to GitHub.* (Когда я закончу дашборд, я загружу его на GitHub. *После WHEN задействован Present Simple*).
